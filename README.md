📊 Análise e Validação de Sistema Completo – Urban Scooter

📌 Descrição

Este projeto tem como objetivo realizar a análise e validação completa de um sistema web e sua API, abrangendo desde a interpretação de requisitos até testes funcionais e validação de dados no back-end.

A abordagem integra testes de interface, lógica de negócio e API, permitindo uma visão ampla do comportamento do sistema e identificação de inconsistências.

🎯 Objetivos

Validar requisitos funcionais e não funcionais

Analisar comportamento da interface e fluxo do usuário

Testar regras de negócio do sistema

Validar endpoints e dados da API

Identificar e documentar falhas

🧠 Metodologia

🔹 Fundamentação teórica

Aplicação de conceitos essenciais como:

Classes de equivalência

Valores limite

Tabela de decisão

Arquitetura cliente-servidor

HTTP e JSON

Boas práticas de reporte de bugs

🔹 Testes de interface (Front-end)

Análise do sistema web com base em requisitos e protótipos no Figma, avaliando o comportamento da aplicação e consistência da interface.

🔹 Testes de API (Back-end)

Validação de endpoints e dados trafegados, garantindo integridade das informações e aderência aos requisitos da API.

🔍 Atividades realizadas

Análise de requisitos do sistema

Criação de mapa mental para estruturação do fluxo

Desenvolvimento de checklists de teste

Elaboração de casos de teste

Validação da interface e funcionalidades

Testes de API e verificação de respostas

Registro e acompanhamento de bugs
